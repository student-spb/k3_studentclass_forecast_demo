# Educational CI/CD Target Service

Этот микросервис используется для демонстрации этапа CI (сборка докер-образа проекта) и деплоя в Kubernetes. 

Статус проекта: **Educational CI/CD Target Service**.

## Архитектура (API Proxy)

Приложение представляет собой легковесный API Proxy на Node.js (Express), который получает запросы от клиентской части, добавляет необходимый токен авторизации (`X-Yandex-Weather-Key`) и делает запрос к Yandex Weather API для получения актуальной информации (блок fact в теле ответа АПИ) о погоде. 

## Локальный запуск

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Убедитесь, что файл `.env` существует и содержит валидный ключ `YANDEX_API_KEY`.

3. Запустите сервер:
   ```bash
   npm start
   ```

4. Локальный запуск и отладка в браузере по URL `http://localhost:3000`.

## Запуск тестов

Проект содержит базовые тесты для:
- проверки ключевой логики (соответствия городов координатам)
- проверки формирования заголовка доступа на API Яндекс Погода
- проверки корректного формирования web-страницы при mock ответа API Яндекс Погода

Тесты завершаются с кодом True при успехе и False при ошибке. Идеально подходит для этапа `Test` в CI/CD конвейерах.

Команда для запуска:
```bash
npm test
```

## CI/CD (GitHub Actions)

В проекте настроен пайплайн непрерывной интеграции (манифест находится в `.github/workflows/ci.yml`). 
Он автоматически запускается при каждом push-событии в ветку `main` и предназначен для сборки артефакта — Docker-образа. 
После успешной сборки готовый образ публикуется в GitHub Container Registry и становится доступен по ссылке: 
`ghcr.io/<ВАШ_GITHUB_ЛОГИН>/weather-app:latest`

## Деплой в Kubernetes

Ниже приведена пошаговая инструкция для деплоя приложения в Kubernetes-кластер.

**Шаг 1:** Создание секрета в кластере командой:
```bash
kubectl create secret generic weather-secrets --from-literal=yandex-api-key="ВАШ_КЛЮЧ_ЯНДЕКСА"
```

**Шаг 2:** Редактирование манифеста. Откройте файл `k8s/weather-app.yaml` и замените `<ВАШ_GITHUB_ЛОГИН>` на ваш реальный логин на GitHub в пути к образу.

**Шаг 3:** Деплой командой:
```bash
kubectl apply -f k8s/weather-app.yaml
```

**Шаг 4:** Проверка выданного порта через команду:
```bash
kubectl get service
```
